## Overview

Pin frequently accessed pages for instant access. Pinned pages always appear at the top of the search modal, separate from recent pages and search results.

## Pinning a Page

### Using Keyboard

1. Open the search modal (**Ctrl+/** or **Cmd+/**)
2. Navigate to a page using arrow keys
3. Press **Space** to pin/unpin

### Using Mouse

Click the bookmark icon on any search result to toggle its pinned state.

## Visual Indicators

| State | Appearance |
|-------|------------|
| **Unpinned** | Outline bookmark icon |
| **Pinned** | Filled purple bookmark icon with background highlight |

The pin icon color can be customized using the `pinColor` theme option or the `--telescope-pin-color` CSS variable.

## Pinned Pages Section

Pinned pages appear in a dedicated section at the top of results:

1. **Pinned Pages** — Your bookmarked pages (manual)
2. **Recently Visited** — Auto-tracked recent pages
3. **Search Results** — Filtered results when searching

## Managing Pins

### Clearing All Pins

Click the **Clear** button next to "Pinned Pages" to remove all bookmarks at once.

### Removing Individual Pins

Press **Space** on a pinned page or click its bookmark icon to unpin it.

## Storage

Pinned pages are stored in `localStorage` under `telescope_pinnedPages` and persist across browser sessions.

## Keyboard Flow Example

```
1. Ctrl+/ → Open modal
2. Type "api" → Filter to API pages
3. ↓ ↓ → Navigate to "API Reference"
4. Space → Pin the page
5. Esc → Close modal

Next time:
1. Ctrl+/ → "API Reference" appears at top!
2. Enter → Navigate instantly
```

## Customizing Pin Appearance

### Via Configuration

```js
starlightTelescope({
  theme: {
    pinColor: '#f59e0b',  // Amber color
  }
})
```

### Via CSS

```css
:root {
  --telescope-pin-color: #ec4899;  /* Pink */
}

.telescope-pin-button.pinned {
  background-color: rgba(236, 72, 153, 0.2);
}
```

## Use Cases

Pinned pages are great for:
- **API Reference** — Quick access to frequently used endpoints
- **Configuration docs** — Options you check often
- **Guides** — Tutorials you're working through
- **Migration docs** — Pages needed during upgrades
